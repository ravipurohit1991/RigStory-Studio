# Phase 1 checkpoint

## Delivered

- Pure domain package [`backend/app/domain`](../../backend/app/domain) with no FastAPI, SQLModel, database, or renderer imports:
  - typed identifiers and factories for project, character, rig, bone, attachment, scene, actor, object, anchor, plan, action, clip, track, keyframe, asset, job, and revision in [`ids.py`](../../backend/app/domain/ids.py);
  - semantic schema versions and comparison in [`versioning.py`](../../backend/app/domain/versioning.py) (`PROJECT_SCHEMA_VERSION = 0.2.0`, `ENGINE_VERSION = 0.1.0`);
  - canonical JSON serialization with stable ordering and NaN rejection in [`canonical.py`](../../backend/app/domain/canonical.py);
  - migration registry with a registered `0.1.0 -> 0.2.0` project migration in [`migrations.py`](../../backend/app/domain/migrations.py);
  - coded validation issues in [`errors.py`](../../backend/app/domain/errors.py).
- 2D math kernel [`backend/app/domain/math2d`](../../backend/app/domain/math2d): `Vec2`, angle normalization and shortest-arc interpolation, `Affine2` matrices (compose, invert, apply), `Transform2D` TRS spec, AABB, polygon helpers, cubic Bezier, and a mulberry32 `SeededRng` with FNV-1a string seeding.
- Core schemas with invariant validators: [`rig.py`](../../backend/app/domain/rig.py) (plus FK evaluation `compute_world_transforms` / `compute_bone_endpoints`), [`character.py`](../../backend/app/domain/character.py), [`scene.py`](../../backend/app/domain/scene.py) (two-actor limit enforced at schema level), [`clip.py`](../../backend/app/domain/clip.py), and the versioned [`project.py`](../../backend/app/domain/project.py) document with migrate-parse-validate loading.
- Mirrored TypeScript kernel [`frontend/src/engine/math`](../../frontend/src/engine/math) and rig validation/FK in [`rig.ts`](../../frontend/src/engine/rig.ts), pinned to the Python kernel by shared golden vectors.
- Zod import schemas [`frontend/src/schemas/project.ts`](../../frontend/src/schemas/project.ts) and cross-object invariants with backend-identical error codes in [`invariants.ts`](../../frontend/src/schemas/invariants.ts).
- Shared fixtures generated by [`generate_fixtures.py`](../../backend/scripts/generate_fixtures.py): two-bone rig, 25-bone canonical biped, empty/one-character/two-character scenes, valid projects ([`biped-demo.rigstory.json`](../../samples/projects/biped-demo.rigstory.json), [`empty-project.rigstory.json`](../../samples/projects/empty-project.rigstory.json)), a legacy `0.1.0` migration fixture, seven invalid fixtures, and [`math-golden.json`](../../samples/fixtures/math-golden.json).
- Sample validator [`validate_project_samples.py`](../../scripts/validate_project_samples.py) now uses the real domain schemas, asserts invalid samples are rejected, and exercises the migration fixture; the CI `project-schema` job installs the backend package accordingly ([`ci.yml`](../../.github/workflows/ci.yml)).
- Developer-only demo page [`DevFixturesPage.tsx`](../../frontend/src/pages/DevFixturesPage.tsx) (dev builds only) that validates the biped demo project in the browser and lists computed world endpoints as text.

## Acceptance criteria evidence

- **Transform composition tests pass with known numerical results.** Backend: `TestAffine2::test_two_bone_composition_known_values` in [`test_math2d.py`](../../backend/tests/domain/test_math2d.py) checks the hand-computed endpoint `(10.370354754472732, 9.69846310392954)`; [`test_rig.py`](../../backend/tests/domain/test_rig.py) checks FK against goldens. Frontend: [`math.test.ts`](../../frontend/src/engine/math/math.test.ts) and [`rig.test.ts`](../../frontend/src/engine/rig.test.ts) verify the same numbers from `math-golden.json`.
- **Property tests confirm `inverse(T) * T` approximately returns identity.** Hypothesis tests `test_inverse_times_self_is_identity`, `test_inverse_round_trips_points`, and `test_composition_is_associative` in [`test_math2d.py`](../../backend/tests/domain/test_math2d.py).
- **Invalid hierarchies detect cycles, missing parents, duplicate IDs, and disconnected bones.** Coded issues `RIG_CYCLE`, `RIG_MISSING_PARENT`, `RIG_DUPLICATE_BONE_ID`, `RIG_DISCONNECTED_BONE` (plus `RIG_NO_ROOT`, `RIG_MULTIPLE_ROOTS`, `RIG_SELF_PARENT`, `RIG_NO_BONES`) are asserted on both sides against the shared invalid fixtures.
- **Canonical JSON round-trips without semantic change.** [`test_canonical.py`](../../backend/tests/domain/test_canonical.py) includes a byte-stable re-serialization of the biped demo document and verifies committed samples are canonical on disk.
- **Sample project files validate in backend and frontend.** Backend: [`test_project.py`](../../backend/tests/domain/test_project.py) and `scripts/validate_project_samples.py`. Frontend: [`project.test.ts`](../../frontend/src/schemas/project.test.ts) parses the same files with Zod and asserts zero invariant issues, and asserts the invalid samples fail.
- **No rendering code is inside the domain math modules.** `backend/app/domain` imports only stdlib and Pydantic; `frontend/src/engine` imports nothing from React, PixiJS, or Fluent.

## Architecture changes

- New pure domain layer under `backend/app/domain` (spec section 7.2); services and API routes do not consume it yet — Phase 2 persistence will.
- New frontend engine kernel under `frontend/src/engine` (math + rig FK) independent of React and rendering, ready for the Phase 2 PixiJS adapter.
- Established the cross-language determinism mechanism: `samples/fixtures/math-golden.json` is generated by the backend and asserted by both test suites.

## Schema changes and migrations

- Project document schema bumped `0.1.0 -> 0.2.0`: `characters`, `scenes`, and `clips` now have full item schemas; `motion_plans` and `generation_records` remain empty-only until Phases 8 and 5.
- Migration `0.1.0 -> 0.2.0` registered in the project migration registry with fixture [`empty-project-0.1.0.rigstory.json`](../../samples/migrations/empty-project-0.1.0.rigstory.json).
- No database schema change; the domain document schemas are file/API contracts, not tables.

## Known limitations

- The frontend import layer accepts only the current schema version; migrating older files in the browser is deferred (backend owns migrations).
- `motion_plans` and `generation_records` are placeholders that only accept empty arrays by design.
- Cross-language float agreement is asserted to 1e-9 for trig-derived values (libm vs V8 may differ in the last ulp); RNG integer sequences are exact.
- Domain models forbid unknown fields; unknown-field preservation currently happens only in the raw-dict migration path.
- The two-actor limit is enforced at parse time; there is no coded `SCENE_TOO_MANY_ACTORS` issue because such a document cannot be constructed.

## Deferred work

- Wiring domain documents into persistence, project CRUD, and revisions (Phase 2.6).
- PixiJS stage adapter and interactive rig editing (Phase 2).
- Clip evaluation/interpolation runtime (Phase 3) — Phase 1 only defines and validates clip schemas.
- Constraint definitions (IK chains, look-at) intentionally left out of `RigDefinition` until Phase 4/7 need them.

## Risks discovered

- Vitest rewrites `import.meta.url` in jsdom, so test fixtures are loaded via `process.cwd()`-relative paths ([`samples.ts`](../../frontend/src/test-utils/samples.ts)); the dev page instead imports through the `@samples` Vite alias.
- Windows Store Python is used locally without a venv; hypothesis was installed user-level. CI installs from `pyproject.toml` (`hypothesis` added to dev extras), so this is a local-environment quirk only.
